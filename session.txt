# ---- Model 2: Linear regression with feature selection ----
# Drop collinear features using VIF, then refit.
import statsmodels.api as sm
from statsmodels.stats.outliers_influence import variance_inflation_factor

def compute_vif(df):
    vif = pd.DataFrame({"feature": df.columns})
    vif["VIF"] = [variance_inflation_factor(df.values, i) for i in range(df.shape[1])]
    return vif.sort_values("VIF", ascending=False).reset_index(drop=True)

X_train_df = pd.DataFrame(X_train, columns=feature_cols)
print("Initial VIF (values >10 indicate problematic multicollinearity):")
print(compute_vif(X_train_df))

# Iteratively drop the highest-VIF feature until all remaining VIFs are < 10.
selected = feature_cols.copy()
while True:
    vif_df = compute_vif(X_train_df[selected])
    worst = vif_df.iloc[0]
    if worst["VIF"] < 10 or len(selected) <= 1:
        break
    selected.remove(worst["feature"])
    print(f"  dropped {worst['feature']} (VIF={worst['VIF']:.1f})")

print(f"\nFinal selected features ({len(selected)}): {selected}")

# Fit using statsmodels for p-values.
X_train_sel = sm.add_constant(X_train_df[selected])
ols = sm.OLS(y_train, X_train_sel).fit()
print("\nOLS summary on selected features:")
print(ols.summary())

# Evaluate on test set.
X_test_sel = sm.add_constant(pd.DataFrame(X_test, columns=feature_cols)[selected])
y_pred_selected = ols.predict(X_test_sel)
r2_selected = r2_score(y_test, y_pred_selected)
rmse_selected = np.sqrt(mean_squared_error(y_test, y_pred_selected))

print(f"\n=== Model 2: Linear + feature selection ({len(selected)} features) ===")
print(f"R²:   {r2_selected:.4f}")
print(f"RMSE: {rmse_selected:.4f} mmHg")

what is rmse, and how good is the model at predicting features, also is a logistic model, or are we predicting the actual vlaues and what does the Rsquared represent in here?
1. RMSE — Root Mean Squared Error

RMSE = √(mean((y_true − y_pred)²))
It's the typical size of your prediction error, in the same units as the target (mmHg).

RMSE = 4.2 mmHg means: on average, your predicted next-window max ICP is off by about ±4.2 mmHg from the truth.

Why squared first? It punishes big misses harder than small ones — being off by 10 is much worse than being off by 1, and RMSE reflects that.

2. Is this logistic or linear?
Linear. You're predicting an actual number (the next window's max ICP value in mmHg), not a yes/no label.

Linear regression → output is a continuous number. "The next max ICP will be 23.4 mmHg."
Logistic regression → output is a probability of a class. "Probability of an ICH event = 0.71."
You could reframe this as logistic by asking "will next_icp_max exceed 20 mmHg? yes/no" — that would make it a classification problem. But your PDF and your ChatGPT conversation both point at regression, so stick with linear. The red 20 mmHg line in the histogram is just context, not a label.

3. What R² means here
R² = "what fraction of the variation in next_icp_max does my model explain?"

R² = 1.0 → perfect prediction, every point on the line
R² = 0.0 → no better than just predicting the mean of y for everything
R² < 0 → worse than predicting the mean (model is actively bad)
Concrete reading for your project:

R²	Interpretation
0.90+	Almost certainly because icp_mean in the past window ≈ icp_max in the next window (ICP is persistent minute-to-minute). Good number, limited insight.
0.60–0.85	Realistic range — model captures the dominant trend but misses spikes.
0.20–0.50	Signal is there but weak; features don't carry enough info.
near 0	Either the target is too noisy or the features are wrong.
4. How good is the model, really?
You'll know once you run the cells, but here's how to judge the numbers when they come out:

Compare RMSE to the standard deviation of the target (regression_data_all["next_icp_max"].std()). If RMSE ≈ std, the model is basically useless — it's no better than guessing the mean. If RMSE is half of std or less, you have real predictive power.
Compare to the trivial baseline of "predict the current window's max ICP as the next window's max ICP" (just persistence). If your linear model can't beat that, the fancy features aren't helping.

# Physiologic ranges (generous for ICU patients):
#   ICP: 0..80 mmHg  (normal 7-15; ICH >20; >60 is terminal)
#   ABP: 30..250 mmHg
#   ECG: -5..5 mV
PHYS_RANGES = {"ICP": (0, 80), "ABP": (30, 250), "ECG": (-5, 5)}

def mask_artifacts(df):
    out = df.copy()
    for col, (lo, hi) in PHYS_RANGES.items():
        out[col] = out[col].where((out[col] >= lo) & (out[col] <= hi))
    return out

What "masking" means
Masking = hiding bad values so they don't pollute your calculations.

Instead of deleting a bad row (which would break the time order of your windows), you replace the bad values with NaN ("not a number"). Then when pandas computes mean, max, or std, it automatically skips the NaNs — as if they were never there.


out[col] = out[col].where((out[col] >= lo) & (out[col] <= hi))
That line reads: "keep the value if it's in range; otherwise replace it with NaN."

Why not just delete bad samples? Because at 50 Hz, a single bump on the ICP transducer might corrupt 20 samples in a row. Deleting them would shift everything around it and the 5-minute window would no longer cover 5 minutes. Masking preserves the time axis.
